# AI 自动评分 Prompt 模板

## 目的与适用范围

本文件是阶段 3 的单一执行入口，用于让 AI 对 `papers_deduped.jsonl` 中的论文进行：

- 相关性评分（1-10 分，保留 1 位小数）
- 子主题标签分配
- 数据抽取字段提取（`design` / `key_findings` / `limitations`）
- 批次结束后的统一回看与自检

默认保持单文件可执行，不要求调用者跨文件拼装 prompt。

## 输入与总体原则

### 输入

- 研究主题：`{topic}`
- 可选范围：`{time_range}`、`{core_question}`
- 论文元数据：`doi/title/abstract/year/venue/authors`

### 总体原则

- 优先基于标题和摘要中的明确证据评分，不要补写不存在的信息。
- 评分、子主题和数据抽取应一次完成，但批次结束后允许统一回看并修正边界样本。
- 输出必须兼容 `scored_papers.jsonl` 的当前下游用法，不要改字段名。
- 如果信息不足以可靠判断，允许输出 `skip: true`，但仍要返回完整 JSON 对象。

## 主题语言建议

- 论文标题和摘要主要为英文时，优先使用英文主题。
- 论文标题和摘要主要为中文时，可以直接使用中文主题。
- 中英混合时，优先使用英文主题或中英双写主题，避免歧义。
- 主题表述应尽量包含研究对象、核心问题和方法线索，而不是只有宽泛领域名。

## 领域说明（天文学/天体物理优先）

- 你是一位学术文献评审专家，精通天文学与天体物理。
- `arXiv astro-ph` 预印本不因“预印本身份”默认降权，应按证据质量、方法严谨性与主题匹配度评分。
- 如主题不属于天文学，也可沿用同一评分框架，但应将“对象/方法/数据”对应到目标领域的实际概念。

## 执行规则

### 评分维度

#### 1. 任务匹配度（Task Alignment）

- 完全匹配：研究相同对象、相同核心科学问题或相同研究目标
- 部分匹配：同一大领域，但对象、机制或目标存在明显差异
- 不匹配：对象或研究问题基本无交集

#### 2. 方法匹配度（Method Alignment）

- 完全匹配：采用相同或高度可比的观测、理论、模拟或分析方法
- 部分匹配：方法路线同属一类，但具体实现不同
- 不匹配：方法完全不同，且结果难以直接互证

#### 3. 数据模态匹配度（Modality Alignment）

- 完全匹配：相同波段、设施、数据集或同类证据来源
- 部分匹配：互补波段、不同设施或相近数据类型
- 不匹配：数据窗口、观测对象或证据形式不具可比性

#### 4. 应用价值（Relevance Value）

- 高：可直接支持综述核心论点、结果比较或方法对照
- 中：可作为背景、互补证据或方法参考
- 低：只有泛泛背景关联

### 分数区间

| 分数区间 | 等级 | 判定原则 |
|---|---|---|
| 9.0-10.0 | 完美匹配 | 相同核心问题，且方法与数据路径高度一致 |
| 7.0-8.9 | 高度相关 | 核心问题一致，但方法或数据略有差异 |
| 5.0-6.9 | 中等相关 | 同一研究领域，但任务、方法或数据差异明显 |
| 3.0-4.9 | 弱相关 | 仅部分概念、方法或技术重叠 |
| 1.0-2.9 | 几乎无关 | 仅背景层面有宽泛关联 |

### 子主题标签规则

- 仅当 `score >= 5.0` 时填写 `subtopic`。
- 当 `score < 5.0` 时，`subtopic` 必须输出空字符串 `""`。
- 优先使用稳定、可合并的大类标签，不要按单一设备、单一事件或单一模型名细分。
- 批次内应主动收敛标签，通常控制在 5-7 个主子主题；如果样本量很小，可少于 5 个，但不要制造一次性标签。

### 核心子主题列表（优先使用）

**观测与样本类**

- `观测约束`
- `统计总体研究`
- `定位与宿主环境`
- `多波段联测`

**理论与模拟类**

- `理论模型`
- `数值模拟`

**方法与资源类**

- `仪器与数据处理`
- `综述`

### 子主题合并规则

- 按科学角色合并，不按具体设备细分：
  - `FAST观测`、`CHIME观测`、`MeerKAT观测` 统一为 `观测约束`
- 按研究问题合并，不按单个模型名细分：
  - `磁星模型`、`吸积模型`、`双星并合模型` 统一为 `理论模型`
- 环境与统计分别归类：
  - `宿主星系`、`局域环境`、`精定位` 统一为 `定位与宿主环境`
  - `事件率`、`红移分布`、`选择效应` 统一为 `统计总体研究`
- 方法学资源统一归类：
  - `搜索算法`、`去色散`、`RFI抑制`、`目录构建` 统一为 `仪器与数据处理`
- 若某个标签在当前批次只出现 1 篇，优先合并到最相近的已有主类。

## 数据抽取规则

### 1. `design`

- 目标：用 5-15 字描述研究设计或方法。
- 优先写观测设施、巡天、理论框架、模拟方法或关键分析手段。
- 避免空泛表述，如“天文研究”“理论分析”。

示例：

- `FAST单脉冲搜索`
- `多波段后随观测`
- `MHD数值模拟`
- `宿主星系光谱分析`

### 2. `key_findings`

- 目标：用 10-30 字概括最重要结果。
- 优先保留摘要中明确给出的数字、样本规模、上限/下限、显著性或参数约束。
- 如果摘要没有可靠数字，可以写非数字结论，但必须忠于摘要，禁止臆造量化值。
- 不要为了满足格式而补写不存在的样本数、显著性或参数。

示例：

- `32个源，事件率上限收紧`
- `偏振特征支持磁星场景`
- `模型可解释重复暴发时变行为`

### 3. `limitations`

- 优先提取摘要中明确写出的局限性。
- 若摘要未明说，可做低风险、保守归纳，如：
  - `样本信息有限`
  - `波段覆盖有限`
  - `独立验证不足`
- 如果无法从摘要做出稳妥判断，输出 `未明确提及`。
- 不要做高风险推断，不要编造作者未暗示的严重缺陷。

## 输出契约

### 字段定义

- `doi: string`
  - 优先输出输入中的 DOI；若输入缺 DOI，则保留原始标识或空字符串，保持与输入一致
- `score: number`
  - 取值范围 `1.0-10.0`，保留 1 位小数
  - 若信息不足而跳过，可输出 `0.0` 并配合 `skip: true`
- `subtopic: string`
  - 当 `score >= 5.0` 时填写子主题
  - 当 `score < 5.0` 或跳过时输出 `""`
- `rationale: string`
  - 用一句话说明打分依据，点明最相关证据
  - 建议 20-50 字，避免泛泛空话
- `alignment.task: "完全匹配" | "部分匹配" | "不匹配"`
- `alignment.method: "完全匹配" | "部分匹配" | "不匹配"`
- `alignment.modality: "完全匹配" | "部分匹配" | "不匹配"`
- `extraction.design: string`
- `extraction.key_findings: string`
- `extraction.limitations: string`
- `skip: boolean`（可选）
  - 仅在标题或摘要缺失、信息明显不足时使用

### 标准 JSON 结构

```json
{
  "doi": "{doi}",
  "score": 8.0,
  "subtopic": "理论模型",
  "rationale": "科学问题一致，采用理论建模解释目标现象，能直接支持主题讨论。",
  "alignment": {
    "task": "完全匹配",
    "method": "部分匹配",
    "modality": "部分匹配"
  },
  "extraction": {
    "design": "辐射机制建模",
    "key_findings": "模型可解释目标源时变行为",
    "limitations": "模型简并明显"
  }
}
```

### 跳过样例

```json
{
  "doi": "{doi}",
  "score": 0.0,
  "subtopic": "",
  "rationale": "标题或摘要缺失，无法可靠评分。",
  "alignment": {
    "task": "不匹配",
    "method": "不匹配",
    "modality": "不匹配"
  },
  "extraction": {
    "design": "",
    "key_findings": "",
    "limitations": ""
  },
  "skip": true
}
```

## 单篇评分 Prompt

```markdown
你是一位学术文献评审专家，精通天文学与天体物理。请评估以下论文与给定研究主题的相关性，并按指定 JSON 结构输出结果。

## 研究主题
- 主题：{topic}
- 时间范围：{time_range}
- 核心问题：{core_question}

## 论文信息
- DOI：{doi}
- 标题：{title}
- 摘要：{abstract}
- 年份：{year}
- 期刊/会议：{venue}
- 作者：{authors}

## 执行要求
- 依据任务匹配度、方法匹配度、数据模态匹配度和应用价值综合打分。
- 分数范围为 1.0-10.0，保留 1 位小数。
- 仅当 score >= 5.0 时填写 subtopic，否则输出空字符串。
- 同步提取 extraction.design / extraction.key_findings / extraction.limitations。
- 如果标题或摘要明显缺失，输出 skip=true 的完整 JSON 对象。
- 只输出 JSON，不要添加任何解释文字。
```

## 批量评分 Prompt

```markdown
请一次性评估以下 {batch_size} 篇论文与主题 "{topic}" 的相关性。

## 输入论文
{papers_jsonl}

## 执行要求
- 对每篇论文输出一个 JSON 对象，字段必须符合本文件“输出契约”。
- 数组元素顺序必须与输入顺序一致。
- 仅当 score >= 5.0 时填写 subtopic。
- 对信息不足的论文，使用 score=0.0 且 skip=true 的完整对象。
- 只输出 JSON 数组，不要添加任何解释文字。
```

## 批次结束后的统一自检

完成一批评分后，请统一回看并按以下顺序检查：

1. **分数区分度**
   - 高分、中分、低分应有明显区分。
   - 目标分布通常可参考：高分约 20-40%，中分约 40-60%，低分约 10-30%。
   - 如果检索本身很精准，允许偏离该分布，但应通过更严格的边界复核避免“普遍虚高”。

2. **子主题收敛**
   - 优先合并相近标签，通常收敛到 5-7 个主子主题。
   - 小批次或小样本时可少于 5 个，但应避免碎片化标签。

3. **边界样本复核**
   - 重点复核 `4.0-7.0` 区间论文，确保没有把弱相关误打成中高分。
   - 检查相似论文的评分是否一致。

4. **跳过项完整性**
   - 凡是 `skip=true` 的条目，都必须保留完整字段结构。

## 注意事项

- 只输出 JSON 或 JSON 数组，不要输出额外解释。
- 不要为满足格式而编造数字、实验结果或局限性。
- 子主题标签服务于后续选文和写作规划，应稳定、可合并、可复用。
- 本文件是阶段 3 的完整执行提示，默认无需再拆成额外模板文件。
