# AI 自动评分 Prompt 模板

## 使用说明

此 Prompt 用于让 AI 直接对文献进行相关性评分，无需调用外部脚本。

## 主题语言建议（重要）

**中文主题 vs 英文主题**：

- **英文主题（推荐）**：
  - ✅ AI 语义理解更准确
  - ✅ 评分区分度更高
  - ✅ 子主题分组更精准
  - 示例：`"Fast radio bursts: origins and emission mechanisms"`

- **中文主题（支持）**：
  - ✅ AI 可直接理解中文主题
  - ⚠️ 可能需要更详细的 Prompt 说明
  - 示例：`"快速射电暴（FRB）起源机制与辐射模型"`

- **主题语言选择原则**：
  - 如果论文标题/摘要主要为英文 → 使用英文主题
  - 如果论文标题/摘要主要为中文 → 使用中文主题
  - 如果混合 → 使用英文主题（通常效果更好）

**脚本评分后备方案**：
- 如果 AI 评分不可用，可使用 `score_relevance.py --method keyword`
- **v3.6 优化**：脚本评分已支持中文主题（自动检测 + 更宽松的评分权重）
- 详见脚本输出的评分分布统计和诊断提示

```markdown

## 论文信息

- **年份**: {year}

## 评分维度（请综合考虑）

### 1. 任务匹配度 (Task Alignment)
- 是否研究相同的天体对象、物理过程或科学问题？
  - 完全匹配：相同源类/相同科学问题/相同研究目标
  - 部分匹配：同一大领域，但对象、机制或目标不同
  - 不匹配：不同源类或不同子领域

### 2. 方法匹配度 (Method Alignment)
- 是否采用可比的观测/理论/模拟/数据分析方法？
  - 完全匹配：相同观测策略、理论框架或模拟方法
  - 部分匹配：同属观测/理论/模拟大类，但具体方法不同
  - 不匹配：方法路线完全不同，且难以直接互证

### 3. 波段/设施/数据匹配度 (Band/Facility/Data Alignment)
- 是否使用相同或可直接比较的观测波段、设施、巡天或数据产品？
  - 完全匹配：相同波段/同类设施/同类数据集
  - 部分匹配：多波段互补或设施不同但问题相近
  - 不匹配：观测窗口、数据类型或适用对象完全不同

### 4. 应用价值 (Relevance Value)
- 对主题研究是否有直接参考价值？
  - 高价值：可直接引用其约束、模型、数据或方法
  - 中等价值：可作为背景、对照或方法参考
  - 低价值：仅有宽泛背景关联

## 评分标准（1-10分，保留1位小数）

> 天文学场景说明：arXiv astro-ph 预印本不因“预印本身份”默认降权，应按证据质量、方法严谨性与主题匹配度评分。

| 分数区间 | 相关性等级 | 描述 | 典型案例 |
|---------|-----------|------|----------|
| **9.0-10.0** | 完美匹配 | 相同科学问题 + 相同方法路径 + 相同数据/波段 | 同一主题下的直接观测约束或核心理论/模拟论文 |
| **7.0-8.9** | 高度相关 | 相同科学问题，但方法、波段或设施略有差异 | 同一对象的互补波段研究或竞争模型研究 |
| **5.0-6.9** | 中等相关 | 同一研究领域，但任务/方法/数据有显著差异 | 同一源类的背景研究、相关统计研究 |
| **3.0-4.9** | 弱相关 | 仅部分概念、方法或技术重叠 | 相近分析技术用于不同天体对象 |
| **1.0-2.9** | 几乎无关 | 仅背景层面有宽泛关联 | 通用算法或无关子领域论文 |

## 子主题标签（请选择最贴切的1-2个）

**重要规则（避免低分污染子主题）**：
- 仅当 **score ≥ 5.0** 时，才需要选择子主题标签
- 若 **score < 5.0**（弱相关/几乎无关），请将 `subtopic` 输出为 `""`（空字符串）

### 核心子主题列表（优先使用）

**观测与样本类**：
- `观测约束` - 基于观测数据给出物理参数、性质或机制约束
- `统计总体研究` - 样本统计、分布、事件率、选择效应
- `定位与宿主环境` - 精定位、宿主星系、局域环境、对应体识别
- `多波段联测` - 射电/红外/光学/紫外/X射线/伽马射线/多信使联合研究

**理论与模拟类**：
- `理论模型` - 起源、辐射、形成、演化等物理模型
- `数值模拟` - MHD/PIC/N体/宇宙学/辐射转移等模拟研究

**方法与资源类**：
- `仪器与数据处理` - 搜寻管线、标定、去噪、RFI抑制、数据产品构建
- `综述` - 综述、白皮书、领域总结

### 子主题合并规则（重要）

为确保子主题收敛到 5-7 个，请遵循以下合并原则：

**规则1：按科学角色合并，不按具体设备细分**
- `FAST观测` + `CHIME观测` + `MeerKAT观测` → **统一用 `观测约束`**
- `光学后随` + `X射线后随` + `射电后随` → **统一用 `多波段联测`**

**规则2：按研究问题合并，不按单个模型名称细分**
- `磁星模型` + `吸积模型` + `双星并合模型` → **统一用 `理论模型`**
- `MHD模拟` + `PIC模拟` + `辐射转移模拟` → **统一用 `数值模拟`**

**规则3：环境与总体统计分别归类**
- `宿主星系` + `局域环境` + `精定位` → **统一用 `定位与宿主环境`**
- `事件率` + `红移分布` + `选择效应` → **统一用 `统计总体研究`**

**规则4：方法学资源统一归类**
- `搜索算法` + `去色散` + `RFI抑制` + `目录构建` → **统一用 `仪器与数据处理`**

**规则5：单例子主题处理**
- 如果某标签只有 1 篇文献，必须合并到最相似的现有子主题
- 例如：`偏振分析`(1篇) → 合并到 `观测约束`（若其主要作用是提供物理约束）

### 最终输出要求

- 子主题数量应控制在 **5-7个** 以内
- 每个子主题至少应有 **2篇文献**
- 优先使用上述"核心子主题列表"中的标签
- 避免创建过于细分的新标签（如 `某望远镜单独结果` → 用 `观测约束`）

### 写作阶段约束（关键）

> **重要**：子主题标签将直接决定综述正文的 `\section` 数量。为确保综述结构清晰、深度充分：

1. **硬性约束**：综述正文除摘要/引言/讨论/展望/结论外，**必须有且仅有 3-7 个子主题段落**
2. **避免过度细分**：不要为单个仪器、单次事件或单个模型名称单独创建 section
3. **合并原则**：
   - 相似观测工作 → 合并为一个大主题（如 "观测约束与样本特征"）
   - 相近物理解释 → 合并（如 "理论模型与辐射机制"）
   - 相近技术路线 → 合并（如 "数据处理与观测方法"）
   - 环境与对应体研究 → 合并（如 "多波段与宿主环境"）

4. **禁止行为**：
   - ❌ 创建 10+ 个子主题 section
   - ❌ 为单一望远镜或单一事件单独创建 section（除非文献量非常大）
   - ❌ 创建过于宽泛的主题（如 "天文学方法"、"数据"）而缺乏具体内容

5. **推荐主题结构示例**：
   - 数据来源与样本构建（巡天、选源、目录、选择效应）
   - 观测特征与经验约束（光变、光谱、偏振、时延等）
   - 理论模型与物理机制
   - 数值模拟与参数空间
   - 多波段/多信使与宿主环境

## 数据抽取表字段提取（与评分同步完成）

> **重要**：在评分的同时，请从摘要中提取以下三个字段，用于生成数据抽取表。这些字段将帮助研究者快速了解每篇文献的核心信息。

### 1. Design（研究设计/方法）

**提取内容**：简洁描述研究方法（5-15字）

**提取规则**：
- **优先级1**：观测设施/巡天/理论框架/模拟方法
- **优先级2**：关键分析手段（如时序分析、光谱拟合、偏振测量、参数反演）
- **优先级3**：数据特点（如多台站、多波段、长时间基线、大样本）
- **优先级4**：任务类型（如源搜寻、定位分析、宿主识别、人口统计）

**示例**：
- "FAST单脉冲搜索"
- "多波段后随观测"
- "MHD数值模拟"
- "辐射转移建模"
- "宿主星系光谱分析"

### 2. Key findings（关键发现）

**提取内容**：核心结果指标（10-30字）

**提取规则**：
- **必须包含**：至少 1 个量化指标、约束量或样本规模
- **优先级1**：样本数、显著性、周期、红移、色散量、光度、能谱、事件率、上限/下限
- **优先级2**：与基线模型或既有结论的比较
- **格式**：逗号分隔，简洁表达

**示例**：
- "<样本数>个源，显著性<σ值>σ"
- "红移<z值>，事件率<数值>"
- "周期<数值>天，重复暴发<次数>"
- "谱指数<数值>，拟合优于基线模型"
- "上限<数值>，排除部分参数空间"

### 3. Limitations（局限性）

**提取内容**：研究局限或不足（5-20字）

**提取规则**：
- **显性局限**：摘要明确提及的不足
  - "样本量有限" / "天空覆盖受限" / "时间基线较短" / "缺乏多波段后随" / "模型简并明显"
- **隐性局限**：基于摘要合理推断
  - 如样本极少 → "样本量有限"
  - 如仅单台设备/单波段 → "波段覆盖有限"
  - 如未提外部交叉验证或独立样本 → "独立验证不足"
- **如无明显局限**：返回"未报告"或"未明确提及"

**示例**：
- "样本量有限"
- "缺乏多波段后随"
- "时间基线较短"
- "模型简并明显"
- "未明确提及"

---

### 提取质量自检

在输出前，请自检：
- ✅ Design 是否包含了具体观测/理论/模拟方法？（而非泛泛的"天文研究"）
- ✅ Key findings 是否至少有 1 个数字或约束量？
- ✅ Limitations 是否基于摘要证据或合理推断？
- ✅ 三个字段是否简洁？（分别控制在 5-15、10-30、5-20字）

请**仅**输出以下 JSON 格式，不要包含任何其他文字：

```json

  "score": <1-10的分数，保留1位小数>,
  "subtopic": "<子主题标签（仅当 score≥5.0 时填写；否则输出空字符串 \"\"）>",

## 评分质量自检 Prompt

```markdown
## 评分质量自检

## 使用流程

1. **单篇评分**：适合论文数量较少（<50篇）或需要高精度场景
2. **批量评分**：适合大量论文（50-200篇），每次处理 20-50 篇
3. **质量自检**：评分完成后必须执行，确保质量

## 技巧与最佳实践

### 1. 渐进式评分
- 第一轮：快速浏览标题，粗分为高/中/低三类
- 第二轮：仔细阅读摘要，精评高/中类文献
- 第三轮：对边界案例（4-7分）重新评估

### 2. 批量处理策略
- 按子主题批量：先处理同一子主题的文献，保持一致性
- 按分数段批量：先评高分段，再评低分段，最后评中间段

### 3. 标注存疑案例
- 对拿不准的文献，在 rationale 中标注"存疑"
- 自检阶段重点重新评估存疑案例

### 4. 保持评分日记
- 在自检时记录评分标准的应用案例
- 为后续一致性提供参考

## 故障排查

### 问题 1：所有文献得分都接近
- **原因**：评分标准应用不一致
- **解决**：重新校准，明确区分度

### 问题 2：子主题过度碎片化
- **原因**：子主题标签过于细分
- **解决**：合并相似子主题，控制在 5-7 个

### 问题 3：高分文献占比过高（>50%）
- **原因**：评分标准过于宽松
- **解决**：提高高分阈值（如从 ≥7 改为 ≥8）

### 问题 4：低分文献占比过低（<10%）
- **原因**：检索策略过于精准，缺少噪声文献
- **解决**：正常现象，说明检索质量高
